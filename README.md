# CDMAProjekt

## Blueprint Learning Platform

Eine plattformübergreifende Lernanwendung auf Basis von **Ionic**, mit der Programmierlogik über ein **Blueprint-/Node-basiertes System** vermittelt wird.

Das Ziel des Projekts ist es, eine interaktive Lernumgebung zu schaffen, in der Nutzer:innen Programmierkonzepte nicht nur theoretisch verstehen, sondern direkt praktisch anwenden können. Dabei sollen Schritt für Schritt Logikabläufe aufgebaut, getestet und nachvollzogen werden.

Ein geplanter Schwerpunkt ist ein **Debugger**, mit dem sich jeder einzelne Ausführungsschritt separat durchspielen lässt. So können Lernende genau verstehen, wie ein Ablauf entsteht, wo Fehler liegen und wie sich Änderungen auf das Ergebnis auswirken.

Als praktisches Lernziel soll es möglich sein, Spiele und Logikrätsel – zum Beispiel **Sokoban** – mithilfe des Blueprint-Systems selbst zu programmieren und bis zum Ziel zu lösen.

## Features

- **Cross-Plattform** dank Ionic
- **Blueprint-/Node-basiertes Programmieren**
- Fokus auf **spielerisches Lernen von Programmierlogik**
- Geplanter **Step-by-Step-Debugger**
- Visualisierung von Abläufen und Programmlogik
- Möglichkeit, kleine Spiele wie **Sokoban** als Lernprojekt umzusetzen

## Vision

Die Anwendung soll besonders für Einsteiger:innen geeignet sein, die Programmierkonzepte visuell und interaktiv lernen möchten.  
Anstatt direkt mit Quellcode zu starten, werden Logik und Kontrollfluss zuerst über Blueprints vermittelt. Dadurch wird der Einstieg in Themen wie:

- Bedingungen
- Schleifen
- Variablen
- Funktionen
- Ereignisse
- Zustandsänderungen

deutlich greifbarer.

## Anwendungsbeispiel

Ein mögliches Lernszenario ist die Entwicklung eines kleinen Spiels wie **Sokoban**:

- Spiellogik mit Blueprints erstellen
- Bewegungen und Kollisionen definieren
- Zielbedingungen überprüfen
- den Ablauf mit dem Debugger Schritt für Schritt analysieren

So lernen Nutzer:innen nicht nur, *dass* etwas funktioniert, sondern auch *warum*.
